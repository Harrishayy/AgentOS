# P1 merge plan — eBPF programs from `Mehul`

**Source:** `origin/Mehul`
**Target:** `integration/p1` (branched from `integration/base`)

## What we want from `Mehul`

- `bpf/` — `common.h`, `*.bpf.c`, `Makefile`
- `policies/` — example policy YAML files
- `iso/README.md`
- `systemd/agentsandbox.service`
- `examples/` — `allowed-net.yaml`, `blocked-net.yaml`, `demo-agent.yaml`, `test-it.sh`
  (will likely be overridden by P3's `examples/` later — keep both for now,
  rename Mehul's collisions with `-mehul.yaml` if needed)
- `Makefile` (top-level, if it adds value beyond `bpf/Makefile`)

## What we DROP from `Mehul`

Per Harrish's integration doc — the daemon and CLI stubs are superseded:

- `daemon/cmd/agentd/`, `daemon/internal/`, `daemon/go.mod`, `daemon/go.sum`
- `cli/agentctl/`

We keep them out at merge time so P2 and P3 land cleanly.

## Conflicts expected

- `setup-vm.sh` — Mehul's version installs eBPF deps; main's is smaller.
  Keep Mehul's.
- `.gitignore` — Mehul's version probably has Go/eBPF entries; merge unioned.
- `Vagrantfile` / `README.md` — keep main's; not changed by Mehul beyond
  cosmetic edits.

## Steps

1. `git checkout -b integration/base origin/main`
2. `git checkout -b integration/p1 integration/base`
3. Cherry-style copy of P1 territory: `git checkout origin/Mehul -- bpf/ policies/ iso/ systemd/ examples/ setup-vm.sh Makefile .gitignore`
   (do not pull `daemon/`, `cli/`, `gui/` from Mehul — those belong to other phases or have been deprecated)
4. If `examples/demo-agent.yaml` already exists on `main`, take Mehul's; if it
   conflicts later with P3's, P3 wins.
5. Commit: `integrate: P1 (Mehul) — eBPF programs, policies, systemd unit`
