## Integration notes (from `arzaan/P4_ORCHESTRATOR.md`)

- Orchestrator talks to P2's daemon over `/run/agent-sandbox.sock`. It's a
  parallel client of the same socket as `agentctl` — does not shell out.
- WebSocket client → P5's relay at `ws://localhost:8765`.
- Event envelope NDJSON: `{"agent": ..., "type": ..., "ts": ..., "data": ...}`.
- P4 manifest is *almost* a subset of P3's (it adds a `mode: enforce|audit`
  field that P3 has not yet specified). We document this gap; we do not
  rewrite the orchestrator — P3 can extend the validator later.

## Conflicts expected

touches. Shared metadata (`README.md`, `Vagrantfile`, etc.) on `arzaan` is
identical to `main` so the checkout is safe.
