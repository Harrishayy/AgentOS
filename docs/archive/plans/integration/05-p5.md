## Integration notes (from P5's `viewer/README.md`)

- Sender role announced via `{role:"sender",name}` first frame.
- Event schemas frozen — must match what P2 (kernel) and P4 (LLM) emit.
- Cross-checked against arzaan's docs: `tool_call.data.args` is a parsed
  object, not a string. Confirmed.

## Conflicts expected

- `viewer/index.html` already exists at `origin/main` (a simple stub). P5
  replaces it with the React build entrypoint and a sibling `viewer-app/`
  directory. Strategy: delete the old `viewer/index.html` before checking
  out P5's tree, OR overwrite via `checkout origin/p5/viewer -- viewer/`.
  The latter is fine because P5 owns the entire `viewer/` directory.
- `.gitignore` — P5 adds entries for `viewer/server/node_modules/` etc.;
  union them into the integration `.gitignore`.
- `CLAUDE.md` — keep P5's (it documents the viewer's working agreement).

## Steps

1. `git checkout -b integration/full integration/p1-p2-p3-p4`
2. `rm -rf viewer/` (drop main's stub) and then
   `git checkout origin/p5/viewer -- viewer/ scripts/setup-git-workflow.sh CLAUDE.md`
